A overloaded template kernel to.

  1. scale input number to 2-bits integers with MSB —> LSB, imag_pol1 -> real_pol1 -> imag_pol0 -> real_pol0
  2. reorder data from [NANT NPOL NTIME] to [NANT NTIME NPOL]

Each thread processes two polarisations at the same time, with this approach we can easily reorder data from [NANT NPOL NTIME] to [NANT NTIME NPOL]

Currently it works with a single coarse channel, multiple antennas and 2 polarisations, we may need to extend it to support multiple coarse channels in future.

Template Parameters
TInput complex data type, which can be half2 or float2

We need to setup the kernel as follow:

[in]muMu of the data of each antenna
[in]sigmaStandard deviation of the data of each antenna
[in]ntimeNumber of time stamps to keep per antenna
[out]outputOutput data in the order og [NANT NTIME NPOL] but with 2-bits each complex component packed to 8-bits integers, each 8-bits integers has data from 2 polarisations

A overloaded template kernel to.

  1. scale input number to required bits (8 and 16 bits, not 4 or 2 bits) of integers
  2. reorder data from [NANT NPOL NTIME] to [NANT NTIME NPOL]

Each thread processes two polarisations at the same time, with this approach we can easily reorder data from [NANT NPOL NTIME] to [NANT NTIME NPOL]

This kernel works with 8 and more bits of output, for 1, 2, 4 bits data, we will probably need a separate kernel (or I need to figure out to merge them).

Currently it works with a single coarse channel, multiple antennas and 2 polarisations, we may need to extend it to support multiple coarse channels in future.

Template Parameters
TINInput complex data type, which can be half2 or float2
TOUTOutput data type, which can be int8_t and int16_t, assumes that no one will use 32 or more bits of integers.
