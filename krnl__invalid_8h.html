CUDA implementation to get generated data frame validity invalid for reverse FFT of selected fine channels.

To setup kernel on host, blockDim.x and gridDim.x are used to define number of output data frames nframe_output, where blockDim.x is number of threads (nthread) per CUDA thread block and gridDim.x is nframe_output/nthread. gridDim.y is number of antennas nant

Kernel dims Value
gridDim.y nant
gridDim.x nframe_output/nthread
blockDim.x nthread
Parameters
[in]nfftNumber of FFTs
[in]nchanNumber of fine channels
[in]nchan_keepNumber of fine channels to keep
[in]nframe_inputNumber of frames per input data stream
[in]nframe_outputNumber of frames per Output data stream
[in]ntime_extraNumber of extra time stamps at buffer sides
[in]nsamp_per_frame_inputNumber of samples per input CODIF data frame.
[in]nsamp_per_frame_outputNumber of samples per output VDIF data frame.
[in]invalid_inputInput frame invalid, which is in the order or [NANT NPOL NFRAME]
[out]invalid_outputOutput frame invalid, which is in the order of [NANT NFRAME_KEEP], where two polarisations are in the same data frame
