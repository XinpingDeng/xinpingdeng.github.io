CUDA implementation of cross correlation with FFT validation markers and delay corrections.

See also
cross
Template Parameters
TThe input data type, which can be cuComplex and half2

It cross correlates input data and accumulates it within the current buffer block, as of that buffer block size determines the length of sub-integration.

input data is channelised data from nant antennas and 2 polarisations, each polarisation has nfft number of FFTs and each FFT has nchan frequency channels.

To setup the kernel on host, blockDim.x and gridDim.x are used to define nchan, where blockDim.x is number of threads (nthread) per CUDA thread block and gridDim.x is 1+(nchan-1)/nthread. gridDim.y and gridDim.z are supposed to be as nant - 1.

The kernel applys fractional sample correlation before cross correlation. It also accumulates cross correlation according to FFT validation marker (which is calculated by

See also
krnl_delay kernel) and scales it with number of validated FFTs in the end.

chan_shifted in this kernel currently assume that we are at odd nyquist zones, otherwise we need to update it as chan_shifted = nchan-1-chan_shifted, so that we can get

  1. right fractional sample corrections
  2. output frequency channels are in ascending order
