A overloaded template kernel to apply FIR to input raw data.

Template Parameters
TINinput complex data type, which can be short2 and uchar2
TOUToutput complex data type, which can be half2 and float2

We need to setup the kernel as follow:

Kernel dims Value
gridDim.z nstream
gridDim.y (nSpectra-1)/SM_COLUMNS+1
gridDim.x nChannels/WARP
blockDim.x THREADS_PER_BLOCK
Parameters
[in]d_dataRaw input data in order [NSTREAM NTIME+EXTRA], where EXTRA is NCHAN*(TAPS-1), NTIME is NFFT*NCHAN
[in]d_coeffFIR coefficients in order [TAPS NCHAN], we may need to mirror it
[in]nSpectraNumber of FFTs, it works when nSpectraSM_COLUMNS != 0
[in]nChannelsNumber of channels, not sure if it works with nChannelsWARP != 0
[out]d_spectraFIR of raw data in [NSTREAM NTIME] order

A overloaded template kernel to merge FIR with unpack.

Template Parameters
TINinput complex data type, which can be short2 and uchar2
TOUToutput complex data type, which can be half2 and float2

We need to setup the kernel as follow:

Kernel dims Value
gridDim.z nstream
gridDim.y (nSpectra-1)/SM_COLUMNS+1
gridDim.x nChannels/WARP
blockDim.x THREADS_PER_BLOCK
Parameters
[in]d_dataRaw input data in order [NSTREAM NTIME+EXTRA], where EXTRA is NCHAN*(TAPS-1) + 2*ntime_extra, NTIME is NFFT*NCHAN
[in]d_coeffFIR coefficients in order [TAPS NCHAN], we may need to mirror it
[in]d_rotatePhase rotation vector in order [NANT NFFT 2] (2 for two numbers, not 2 polarisations), all 0 if we do not want to rotate it
[in]d_shiftShifts in samples in oder [NANT NFFT], all 0 if we do not want to shift it
[in]ntime_extraExtra time stamps at each end
[in]nSpectraNumber of FFTs, it works when nSpectraSM_COLUMNS != 0
[in]nChannelsNumber of channels, not sure if it works with nChannelsWARP != 0
[out]d_spectraFIR of raw data in [NSTREAM NTIME] order
